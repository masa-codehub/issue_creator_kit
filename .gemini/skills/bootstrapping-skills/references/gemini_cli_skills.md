# エージェントスキルの作成

このガイドでは、Gemini CLI の機能を拡張するための独自の「エージェントスキル」を作成する方法について概要を説明します。

## はじめに: `skill-creator` スキルの利用

新しいスキルを作成する推奨方法は、組み込みの `skill-creator` スキルを使用することです。これを使用するには、Gemini CLI に新しいスキルの作成を依頼してください。

**プロンプト例:**

> "create a new skill called 'code-reviewer'"
> （'code-reviewer' という新しいスキルを作成して）

Gemini CLI は `skill-creator` を使用して以下のようにスキルを生成します:

1.  スキル用の新しいディレクトリを生成します (例: `my-new-skill/`)。
2.  必要な YAML フロントマター (`name` および `description`) を含む `SKILL.md` ファイルを作成します。
3.  標準的なリソースディレクトリ (`scripts/`, `references/`, `assets/`) を作成します。

## 手動でのスキル作成

手動でスキルを作成したい場合は、以下の手順に従います:

1.  スキルのための **ディレクトリを作成** します (例: `my-new-skill/`)。
2.  新しいディレクトリ内に **`SKILL.md` ファイルを作成** します。

スキルをサポートする追加リソースを加える場合は、「スキルの構造」セクションを参照してください。

## スキルの構造

スキルとは、ルートに `SKILL.md` ファイルを含むディレクトリのことです。

### フォルダ構成

必須コンポーネントは `SKILL.md` ファイルのみですが、スキルのリソースを整理するために以下の構造を推奨します:

```text
my-skill/
├── SKILL.md       (必須) 指示書およびメタデータ
├── scripts/       (任意) 実行可能なスクリプト
├── references/    (任意) 静的なドキュメント・仕様書
└── assets/        (任意) テンプレートやその他のリソース
```

### `SKILL.md` ファイル

`SKILL.md` ファイルはスキルの中核です。このファイルでは、メタデータに YAML フロントマターを使用し、指示内容には Markdown を使用します。例:

```markdown
---
name: code-reviewer
description: コードレビューを行うためのスキルです。ローカルの変更とリモートの Pull Request の両方をサポートします。
---

# Code Reviewer

このスキルは、エージェントが徹底的なコードレビューを行うためのガイドを提供します。

## ワークフロー

### 1. レビュー対象の決定

- **リモート PR**: ユーザーが PR 番号または URL を指定した場合、そのリモート PR を対象とします。
- **ローカルの変更**: 変更がローカルにある場合... ...
```

- **`name`**: スキルの一意な識別子です。これはディレクトリ名と一致させる必要があります。
- **`description`**: スキルが何を行い、Gemini がいつそれを使用すべきかを説明します。
- **Body (本文)**: ファイルの Markdown 本文には、スキルがアクティブなときにエージェントの振る舞いをガイドする指示が含まれます。
