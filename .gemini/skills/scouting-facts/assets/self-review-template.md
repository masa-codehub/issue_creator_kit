# 自己レビュー (Self-Review)

## 1. 客観性 (Objectivity)
- [ ] **事実の分離:** 「〜と思われる」といった主観や提案を排除し、事実のみを記述しているか？
  - **根拠:** (主観的な表現を排除した具体的な箇所や工夫を記述)
- [ ] **推測の排除:** 不確かな情報を断定的に書いていないか？
  - **根拠:** (情報の確実性をどう担保したか記述)

## 2. 追跡可能性 (Traceability)
- [ ] **根拠の明示:** 全ての事実に対して、ファイルパスや行番号が示されているか？
  - **根拠:** (主要な事実と対応するパスの例を記述)
- [ ] **コード引用:** 重要なロジックについては、実際のコード断片が引用されているか？
  - **根拠:** (どのファイルのどのロジックを引用したか記述)

## 3. 網羅性 (Completeness)
- [ ] **範囲のカバー:** 定義した調査範囲を漏れなく調査したか？
  - **根拠:** (調査範囲として定義した項目すべてに触れていることを確認)
- [ ] **否定の証明:** 「存在しないこと」や「記述がないこと」も重要な事実として記録されているか？
  - **根拠:** (記述がなかったことにより判明した事実を記述)

## 4. 分析フェーズへの適合性 (Input Quality for Analysis)
- [ ] **意図とのリンク:** ユーザーの依頼キーワードに関連する情報（肯定的・否定的問わず）が含まれているか？
  - **根拠:** (依頼文のどの単語に対応する調査結果か記述)
- [ ] **制約の明示:** 後続の分析で「実現可能性」や「リスク」を判断するための材料（制約、依存関係、技術的負債）が明記されているか？
  - **根拠:** (特定した主要な制約や矛盾点を記述)
- [ ] **未確定事項:** 調査しきれなかった点が、推測ではなく「未調査」として記録されているか？
  - **根拠:** (あえて調査を打ち切った、または不明だった項目を記述)
