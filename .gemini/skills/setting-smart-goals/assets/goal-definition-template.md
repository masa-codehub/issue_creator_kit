# 目標定義書 (Goal Definition)

## 1. 核心的目標 (Core Intent / SMART Goal)
<!--
【具体的(S) / 関連性(R)】
何を達成し、それによってどのような価値（Outcome）を生むのか。
分析レポートのどの仮説を採用したかを明記する。
-->
- **ゴール:** (例: 認証トークンの有効期限を24時間に変更し、テストをパスさせる)
- **採用した仮説:** (例: 案A - 定数定義の修正とリグレッションテスト)
- **期待される価値:** (例: ユーザーの利便性向上と、仕様と実装の不一致の解消)

## 2. 実行の前提条件 (Achievable / Prerequisites)
<!--
【達成可能(A)】
作業開始時にこの状態であることを保証する。
-->
- **対象ファイル:** (例: `src/config.py`, `tests/test_auth.py`)
- **必要な情報:** (例: 現在の `EXPIRATION` の値、既存テストの実行結果)
- **依存タスク:** (例: なし)

## 3. アクションプラン (Specific / Time-boxed)
<!--
【具体的(S) / 期限(T)】
1ターンの応答で完遂できる具体的なステップ。
-->
1. **[確認]:** (例: 現状の定数とテスト失敗を再確認)
2. **[変更]:** (例: `EXPIRATION` を `86400` に修正)
3. **[検証]:** (例: `pytest` で修正箇所をピンポイントで検証)

## 4. 完了定義 (Measurable / Definition of Done)
<!--
【測定可能(M)】
主観を排した機械的な合格基準。
-->
### A. 自動検証 (Automated)
- **検証コマンド:**
  ```bash
  # (例: pytest tests/unit/test_auth.py::test_token_expiry)
  ```
- **合格基準:** (例: `1 passed` が出力されること)

### B. 状態検証 (State Check)
- **確認対象:** (例: `src/config.py` の内容)
- **合格基準:** (例: `EXPIRATION = 86400` という文字列が含まれていること)

## 5. 制約と安全策 (Constraints & Safety)
- **負の制約:** (例: `shared` パッケージ内の共通クラスには手を加えない)
- **安全策:** (例: 変更前に元のファイルのバックアップをメモリに保持する)

## 6. SMART 自己評価
- **Specific:** 変更箇所とアクションがファイル単位で特定されているか？
- **Measurable:** コマンド一つで成功を判定できるか？
- **Achievable:** 必要なファイルの内容は調査済みか？
- **Relevant:** ユーザーの真の目的を解決しているか？
- **Time-boxed:** 1ターンの作業として適切な量か？
