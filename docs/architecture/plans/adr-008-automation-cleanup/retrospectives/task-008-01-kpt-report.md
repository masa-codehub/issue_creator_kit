# Retrospective Report (KPT) - Archive ADR-003 Docs

## 1. Keep (継続すべきこと)

- **現状把握の徹底:** `git status` により、自分の作業開始前に既に行われていた変更（renamed & staged）を即座に把握し、二重作業を防いだ。
- **SMARTゴールの活用:** `defining-work-goals` を通じて、リンク修正の不備（2/4ファイルのみ修正済み）を具体的に特定し、明確な完了条件を立てることができた。
- **成果物の局所性:** `GEMINI.md` の原則に従い、計画ディレクトリ内に `reviews/` を作成して分析・監査レポートを集約した。

## 2. Problem (課題と原因)

- **ツールの誤用:** `run_shell_command` 内で `read_file` を呼び出すという、ツール間の境界を混同するミスが発生した。
- **コンテキストの不一致:** ユーザープロンプトのディレクトリ構造と実際のファイルシステムの状態（既に移動済みだがコミットはされていない）の乖離により、最初の調査でファイルが見つからないエラーが発生した。

## 3. Try (次へのアクション)

- **実証的改善:** ファイル移動を伴うタスクでは、移動後の相対パス調整をチェックリストに明文化し、今回のように一括置換で漏れなく対応する。
- **仕組みの改善:** アーカイブ化の際に共通で使用できる「DEPRECATED」テンプレートを `assets/` に用意し、一貫性を高める。

## 4. 4 Axes Evaluation

- **Safety (安全性):** 削除ではなく `git mv` を行い、履歴を保護した。
- **Efficiency (効率性):** 既にステージングされていた作業を活かし、最小限の修正で完了させた。
- **Context (コンテキスト):** 4ファイルすべての状態を揃え、最新設計への正しい誘導リンクを提供した。
- **Alignment (整合性):** ADR-008 のクリーンアップ方針と完全に一致している。
