# Analysis Report - Archive Obsolete Architecture Docs

## 1. 意図の深掘り (Intent Deep Dive)
- **課題:** ADR-003 時代の古い設計書がルートディレクトリに残っており、最新の ADR-007/008 と混同されるリスクがある。
- **目的:** 物理配置をクリーンにし、歴史的経緯の保存と最新情報の明確化を両立させる。
- **価値:** 開発者の認知負荷低減と、設計情報の整合性維持。

## 2. ギャップ分析 (Gap Analysis)
- **現状:** `git mv` 自体は実行済みでステージングされている。
- **不足:** 一部のファイル (`arch-state-003-task-lifecycle.md`, `arch-structure-003-vqueue.md`) において、移動後の相対パス調整が完了していない可能性がある。
- **制約:** 削除は禁止（`git mv` を維持）。

## 3. 仮説の立案 (Hypotheses)

### 3.1. 実証的仮説 (Grounded) - 本命案
- **内容:** ステージング済みのファイルを尊重しつつ、未修正の2ファイルに対してパス調整（`../` の追加等）と最終確認を行い、コミットする。
- **検証:** `grep` 等で相対パスの妥当性を確認。

### 3.2. 飛躍的仮説 (Leap)
- **内容:** アーカイブされたドキュメント間で相互リンクがある場合、それらもすべて `./` から `./` (同一ディレクトリ内) または適切な相対パスに一括変換し、アーカイブ内での閲覧性を担保する。

### 3.3. 逆説的仮説 (Paradoxical)
- **内容:** リンク切れを「許容」するという制約を利用し、あえて複雑なパス調整は行わず、警告文の追加のみに留めて作業時間を最小化する。しかし、今回は既に2ファイルが修正済みであるため、一貫性を保つべき。

## 4. 推奨アプローチ
- 実証的仮説を採用。残りの2ファイルのリンクを修正し、4ファイルすべての状態を揃えた上でコミットを提案する。
