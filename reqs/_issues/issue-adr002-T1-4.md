- **Task ID**: T1-4

## 1. 目的と背景
Phase 2 の実装品質を保証するためのテストケースを事前に定義します。
「何を以て完了とするか」をテストコードの仕様として明確にします。

## 2. 実装指示 (Implementation Instructions)
- [ ] T1-3 で設計した関数に対するテストシナリオを網羅的にリストアップせよ。
- [ ] **`update_metadata` のテストケース**:
    - **正常系**:
        - 単一キーの更新（Status のみ）
        - 複数キーの同時更新（Status と Date）
        - 既存の値が空の場合の更新
    - **異常系・準正常系**:
        - 指定したキーが存在しない場合（無視するか、追記するかの方針決定含む）
        - ファイルが空の場合
    - **境界値・エッジケース**:
        - キーの前後に余分な空白がある場合
        - Markdown のリスト記法（`-`, `*`）が混在している場合
- [ ] テストデータ（入力Markdownと期待される出力Markdownのペア）を具体的に作成せよ。

## 3. 検証基準 (Definition of DoD)
- [ ] テストケース一覧が作成されていること。
- [ ] 各ケースに対し、具体的な入力と期待値が定義されていること。

## 4. 影響範囲と注意事項
- テストコードの実装（`pytest` の記述）自体は Phase 2 で行うが、ここでは「どのようなテストコードを書くか」を自然言語または疑似コードで定義する。
